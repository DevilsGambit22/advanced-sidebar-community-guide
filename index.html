How to Create an Advanced Chess.com Club Sidebar

A complete beginner’s guide to building an original sidebar, publishing it free with GitHub Pages, and embedding it into a Chess.com club forum or announcement.

No coding experience required GitHub Pages Mobile-friendly Original designs only
Original work only: This guide teaches the development process. It does not grant permission to copy another club’s branding, graphics, writing, music, private code, layout, or distinctive features.
Part 1 — Understand the Files
index.htmlThe structure and visible content.
style.cssThe colors, spacing, borders, layout, and animations.
script.jsOptional interactive behavior.

GitHub Pages hosts static files. That means a sidebar can use HTML, CSS, JavaScript, and images without requiring a traditional web server.

Part 2 — Create Your Project Folder
  1. Create a folder named club-sidebar.
  2. Create index.html, style.css, and script.js inside it.
  3. Create an assets folder for your own images.
  4. Keep filenames short, clear, and lowercase.
club-sidebar/
├── index.html
├── style.css
├── script.js
└── assets/
    ├── club-logo.png
    └── hero-image.jpg
Filename warning: GitHub treats uppercase and lowercase letters as different. Logo.png and logo.png are different files.
Part 3 — Build the HTML

Paste this into index.html. Replace the placeholder club name, description, image filename, and links with your own information.

<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>My Chess Club Sidebar</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <main class="sidebar">
    <section class="hero">
      <img src="assets/club-logo.png" alt="My club logo">
      <h1>My Chess Club</h1>
      <p>Welcome to our club community.</p>
    </section>

    <section class="panel">
      <h2>Quick Links</h2>
      <a class="button"
         href="https://www.chess.com/club/YOUR-CLUB"
         target="_blank"
         rel="noopener noreferrer">
        Visit Our Club
      </a>
    </section>

    <section class="panel">
      <h2>About Us</h2>
      <p>Add a short, welcoming description of your club here.</p>
    </section>
  </main>

  <script src="script.js"></script>
</body>
</html>
File paths: assets/club-logo.png means “open the assets folder and load club-logo.png.” The spelling and capitalization must match exactly.
Part 4 — Add the Design

Paste this into style.css. It creates a clean responsive layout that works inside a narrow embedded area.

* {
  box-sizing: border-box;
}

html,
body {
  margin: 0;
  min-height: 100%;
  background: #151515;
  color: #f2f2f2;
  font-family: Arial, sans-serif;
}

body {
  overflow-x: hidden;
  overflow-y: auto;
}

.sidebar {
  width: 100%;
  max-width: 760px;
  margin: 0 auto;
  padding: 16px;
}

.hero,
.panel {
  margin-bottom: 16px;
  padding: 20px;
  background: #242424;
  border: 1px solid #494949;
  border-radius: 14px;
  text-align: center;
}

.hero img {
  display: block;
  width: 110px;
  height: 110px;
  margin: 0 auto 12px;
  border-radius: 50%;
  object-fit: cover;
}

.button {
  display: block;
  margin: 10px 0;
  padding: 12px 15px;
  color: #ffffff;
  background: #3c3c3c;
  border: 1px solid #676767;
  border-radius: 9px;
  text-decoration: none;
}

.button:hover {
  background: #4a4a4a;
  transform: translateY(-2px);
}

@media (max-width: 600px) {
  .sidebar {
    padding: 10px;
  }

  .hero,
  .panel {
    padding: 16px;
  }
}
Iframe-friendly: The page uses responsive widths, allows vertical scrolling, avoids horizontal overflow, and includes a mobile layout.
Part 5 — Add Optional JavaScript

Your first sidebar does not require JavaScript. You may leave script.js empty. This harmless example updates an element with the ID year:

const yearElement = document.getElementById("year");

if (yearElement) {
  yearElement.textContent = new Date().getFullYear();
}
Security: Never place passwords, private API keys, personal access tokens, or sensitive personal data in a public GitHub repository.
Part 6 — Test the Sidebar Before Publishing
  1. Save all project files.
  2. Open index.html in your browser.
  3. Check every image, button, and link.
  4. Narrow the browser window to test mobile sizing.
  5. Confirm the page scrolls vertically and does not scroll sideways.
Your final test should use the published GitHub Pages URL because some browser behavior differs when a file is opened directly from a computer.
Part 7 — Upload the Files to GitHub
  1. Create or sign in to a GitHub account.
  2. Choose New repository.
  3. Name it something simple, such as my-club-sidebar.
  4. Use a public repository for the normal free GitHub Pages workflow.
  5. Select Add file → Upload files.
  6. Upload all files while preserving the assets folder.
  7. Commit the changes.
Part 8 — Enable GitHub Pages
  1. Open the repository’s Settings.
  2. Select Pages.
  3. Under Build and deployment, choose Deploy from a branch.
  4. Select main and /(root).
  5. Select Save.
  6. Open the published URL shown by GitHub after deployment.
https://YOUR-USERNAME.github.io/YOUR-REPOSITORY/
Part 9 — Embed the Published Page on Chess.com
  1. Open a club forum or announcement editor where you have permission to post.
  2. Use the editor’s media/embed option.
  3. Paste an iframe containing your GitHub Pages URL.
  4. Preview the post on desktop and mobile before publishing.
<iframe
  src="https://YOUR-USERNAME.github.io/YOUR-REPOSITORY/"
  title="My Chess Club Sidebar"
  width="100%"
  height="760"
  scrolling="yes"
  loading="lazy"
  style="display:block;width:100%;max-width:100%;border:0;overflow:auto;"
></iframe>
Chess.com note: Editor capabilities may change, and not every posting area necessarily accepts the same embed markup. Use the media/embed interface rather than pasting iframe code as ordinary visible text.
Part 10 — Update the Sidebar Later
  1. Edit the existing file in the same GitHub repository.
  2. Commit the change to the branch used by GitHub Pages.
  3. Keep the repository name and public URL unchanged.
  4. Refresh the embedded page after the new deployment finishes.

You normally do not need to replace your Chess.com iframe when the hosted URL remains the same.

Troubleshooting
  • 404: Check the Pages source branch, root folder, and location of index.html.
  • Broken image: Verify the path, spelling, capitalization, and file extension.
  • Old content: Refresh after deployment or test in a private browser tab.
  • Content cut off: Increase iframe height and allow vertical scrolling.
  • Sideways scrolling: Use responsive widths, responsive images, and overflow-x:hidden.
  • External link opens inside the embed: Use target="_blank" and rel="noopener noreferrer".
Final Checklist
  • The main file is named exactly index.html.
  • All content and images are original or properly licensed.
  • No secrets or private credentials are stored in GitHub.
  • Every link and image has been tested.
  • The design works on desktop and mobile.
  • The GitHub Pages URL works before embedding.
  • The Chess.com post has been previewed before publishing.
Finished: You now understand how to build, publish, embed, and maintain an original advanced club sidebar.
